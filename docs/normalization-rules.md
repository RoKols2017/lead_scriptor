# Правила нормализации

## Базовые принципы

- Нормализуем консервативно.
- Предпочитаем пустую строку или пустой массив выдуманному содержимому.
- Неоднозначный ввод помечаем как weak, если его нельзя подтвердить.
- Provenance с типом `inferred` всегда остается явным.

## Слабые паттерны

- `для всех`
- `любой бизнес`
- `автоматизирует все`
- `дешево`, `быстро`, `качественно` без измеримой конкретики

## Пояснения по полям

- `product.market_type` принимает только `B2B`, `B2C`, `B2B2C`.
- `sales.auto_discover_titles=true` допустим только когда должности неизвестны, но тип компании уже известен.
- Поля `tone` влияют на styling, но не переписывают подтвержденные бизнес-факты.
