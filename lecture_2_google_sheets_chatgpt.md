## 1. Кастдев с помощью ChatGPT в Google Sheets

Работая над этой задачей, мы не будем углубляться в методологию кастдева, а будем использовать её лишь как пример.

### План работы над задачей

В данном примере реализован многоступенчатый процесс промпт-инжиниринга, где последовательно подаются запросы, и каждый последующий результат зависит от предыдущего.

⬇️ Сгенерируем описание продукта.  
⬇️ На основе описания продукта выделим ядро целевой аудитории.  
⬇️ Проведем сегментацию целевой аудитории.  
⬇️ Сгенерируем подробное описание сегмента.  
⬇️ На основе описания сегмента сгенерируем описание аватара, с которым будем проводить кастдев.  
⬇️ Сформулируем гипотезу.  
⬇️ Сгенерируем вопросы для кастдева.  
⬇️ Проведем кастдев.

Первым шагом нам необходимо сгенерировать подробное и структурированное описание продукта, которое можно использовать для различных маркетинговых и аналитических целей. В данном случае оно нужно для подготовки к проведению кастдева.

### Придумываем гипотезу

#### ⚙ Промпт

`СЦЕПИТЬ("Я хочу сформулировать гипотезы для моего продукта

Представь, что ты - эксперт по формулировке гипотез. Ты обладаешь глубокими знаниями о том, как создавать гипотезы, которые являются: Четкими и ясными, Измеримыми, Релевантными, фальсифицируемыми Логически обоснованными. Ты также обладаешь опытом работы с различными типами гипотез, в том числе: Гипотезы о продукте, Гипотезы о целевой аудитории, Гипотезы о маркетинге. Ты можешь помочь людям сформулировать гипотезы, которые помогут им получить информацию, необходимую для принятия обоснованных решений о своем бизнесе.

Критерии формулировки гипотезы:

- Четкость и ясность - Гипотеза должна быть сформулирована ясно и понятно, чтобы не возникало разночтений. Избегайте двусмысленности и неопределенных терминов.
- Ориентация на решение проблемы - Гипотеза должна быть направлена на решение конкретной проблемы, с которой сталкивается целевая аудитория.
- Измеримость - Гипотеза должна быть измеримой, то есть вы должны иметь возможность проверить ее с помощью данных.
- Фальсифицируемость - Гипотеза должна быть фальсифицируемой, то есть должна существовать возможность ее опровергнуть.
- Релевантность - Гипотеза должна быть релевантна вашей бизнес-цели.
- Фокус на одной переменной - В идеале, гипотеза должна фокусироваться на одной переменной, чтобы можно было четко определить влияние этой переменной на результат.
- Учет целевой аудитории - Гипотеза должна учитывать характеристики и потребности вашей целевой аудитории.
- Логическая обоснованность - Гипотеза должна быть логически обоснованной и иметь под собой рациональное объяснение.
- Возможность проверки - Гипотеза должна быть сформулирована таким образом, чтобы ее можно было проверить с помощью доступных вам методов исследования.

Дополнительные рекомендации:

Используйте простые и понятные слова.

Избегайте жаргона и технических терминов.

Сформулируйте гипотезу как можно более конкретно.

Убедитесь, что ваша гипотеза не является очевидной.

Формат <Если ..., то ...> - Гипотеза должна быть сформулирована в формате <Если ..., то ...>, где <если> - это условие, а <то> - это ожидаемый результат.

Примеры Формата <Если…, то…>:

<Если мы сделаем приложение для медитации более доступным, то больше людей будут им пользоваться.>

<Если мы добавим больше онлайн-курсов для предпринимателей, то мы сможем увеличить нашу выручку.>

<Если мы будем продвигать наши экологически чистые продукты для дома в социальных сетях, то мы сможем повысить узнаваемость бренда.>

<Если мы предложим бесплатную доставку, то количество заказов увеличится на 10%.>

<Если мы изменим дизайн сайта, то конверсия увеличится на 5%.>

<Если мы запустим рекламную кампанию в социальных сетях, то узнаваемость бренда увеличится на 20%.>

## Бонус: Использование ключей API от сервиса Proxyapi при отсутствии платной версии ChatGPT

В этом видео мы расскажем, как добавить свой API-ключ в таблицу и работать с формулами, не используя расширения.

Перед тем как приступить к работе с таблицей, создайте API-ключ, скопируйте его и переходите к следующей части текста.

### Подключаем ключ в Google таблице

1. Создадим в таблице новый лист и копируем на него ключ.
2. Подключаем ключ. Вверху таблицы на панели инструментов заходим во вкладку **Расширения → Apps Script**. Открывается окно создания скрипта, который мы будем использовать.
3. В открывшемся окне удаляем ненужные данные, чтобы рабочая область была чистой. Вставляем код Java Script.

> 💡 Эксперт использует код с API-ключом от ProxyApi, но ты можешь использовать код для работы с API-ключом напрямую от OpenAI.

### Код с ProxyApi

```javascript
function chatGPT(
prompt,
apikey = "Введи свой ключ API",
max_tokens,
temperature = 0.7,
model = "gpt-3.5-turbo"
) {

const url = "https://api.proxyapi.ru/openai/v1/chat/completions";
const payload = {
model,
max_tokens,
temperature,
messages: [{ role: "user", content: prompt }]
};

const options = {
contentType: "application/json",
headers: { Authorization: "Bearer "+ apikey },
payload: JSON.stringify(payload)
};

const res = JSON.parse(UrlFetchApp.fetch(url, options).getContentText());
return res.choices[0].message.content.trim();
}
```

### Код с API от OpenAI

```javascript
function chatGPT(
  prompt,
  apikey = "Введи свой ключ API", // Убедитесь, что вы вставили ваш ключ API
  max_tokens = 100,
  temperature = 0.7,
  model = "gpt-3.5-turbo"
) {

  // URL OpenAI API для выполнения запросов
  const url = "https://api.openai.com/v1/chat/completions";

  // Структура payload для запроса
  const payload = {
    model: model,
    max_tokens: max_tokens,
    temperature: temperature,
    messages: [{ role: "user", content: prompt }]
  };

  // Опции запроса
  const options = {
    method: "POST",
    contentType: "application/json",
    headers: {
      Authorization: "Bearer " + apikey // Использование API-ключа для авторизации
    },
    payload: JSON.stringify(payload) // Преобразуем тело запроса в формат JSON
  };

  // Отправка запроса и получение ответа
  const res = JSON.parse(UrlFetchApp.fetch(url, options).getContentText());

  // Возвращаем ответ от модели
  return res.choices[0].message.content.trim();
}
```

4. Сохраним изменения. Наверху страницы в строке с инструментами нажмём значок с дискетой.
5. Переименуем проект. Вверху страницы отображается название. Сейчас это: **Проект без названия**. Кликнем на него. Открывается окошко с названием. Назовем проект **ChatGPT**. Нажимаем **Переименовать**.
6. Наверху страницы в строке с инструментами нажимаем **Выполнить → Проверить разрешения**.
7. Попадаем на страницу подтверждения аккаунта. Выбираем свой аккаунт.
8. В открывшемся окне заходим в **Дополнительные настройки**. Выбираем перейти на страницу ChatGPT (небезопасно).
9. В следующем открывшемся окне нажимаем **Разрешить**.
10. Нас возвращает на страницу Apps Script, где выполняется запрос.

Важно: журнал выполнения выдаёт ошибку. Не обращайте внимания, так и должно быть.

11. Возвращаемся к Google таблице.
12. Сформируем запрос с помощью подключенного ключа по следующей схеме: `= chatgpt (A1; ’Лист 2’ А1)` и нажимаем Enter.

В ячейке А1 у нас написано: напиши подводку к данной новости “Apple анонсировали внедрение ИИ в свои операционные системы”

Лист 2 в формуле — ссылка на ячейку с ключом на втором листе таблицы.

Чтобы запрос сработал нужно пополнить кошелёк Proxyapi. При нулевом балансе в ячейке будет ошибка.

Важно: при работе с ключом от Proxyapi ссылка на ключ должна быть в каждом запросе.

Обратите внимание, что при вызове через ячейку формулы `chatgpt` подсвечиваться она не будет. Поскольку мы используем прослойку — специальный код JavaScript. Прослойка отправляет наш запрос в Proxyapi, Proxyapi перенаправляет его в ChatGPT, получает ответ и направляет нам в Google таблицу.
